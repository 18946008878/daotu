# daotu
极简实用的导游收支记录器，专攻「支出项」和「收入项」的快速记录与统计，特别突出导游业务特色

## 精简版核心功能规划：快、准、专

### 1. 核心定位
* 只解决一件事：清晰记录带团过程中的每一笔支出和每一笔收入，并自动按行程汇总。

### 2. 核心模块（极简 MVP)
* A. 行程管理（骨架）
  * 创建行程：只需输入：
    * 团号、名称（必填，如 “0625-北京长城一日游”）
    * 日期（必填，带团日期）
    * 备注（可选，如 旅行社名称、人数）
  * 行程列表：按日期倒序排列，清晰展示每个团。

      * 自费（应用输入框添加游客自愿参加额外项目，如表演、加景点。有收入金额，支出金额，用收入金额减去支出金额自动计算利润等） - **导游核心收入2**

### 3. 核心统计（一目了然）
* 行程总览（每个行程的核心页面）：
  * 总收入：醒目大字显示该团所有收入总和。
  * 总支出：醒目大字显示该团所有支出总和。
  * 净收入（利润）：最核心指标！总收入 - 总支出，用显著颜色（如盈利绿色，亏损红色）超大字体显示。
  * 收支明细列表：合并展示该行程下按时间排序的所有支出和收入记录。

### 4. 极简设置
* 数据备份、导出（基础）：支持将单个行程或所有数据导出为 Excel/CSV文件。本地存储优先。

### 5. 核心设计原则（快字当头）
* 主界面 = 行程列表：打开App
* 3步完成记录：选行程 → 点 ·+支出·、·+收入· → 选分类+输金额（备注可选）→ 完成。10秒内搞定一笔！
* 大按钮、大字体：方便户外、移动中操作。
* 离线优先：所有操作不依赖网络，数据存手机本地。
* 零学习成本：界面极其直观，不用看说明书。

### 6. 技术栈建议（更轻量）
* 移动端：
  * Android: Flutter（跨平台）。
* 数据库：SQLite（本地存储）足够，无需后端服务器。
* 核心库：本地数据库操作

### 7. MVP开发路线图（快速上线）
1. 核心骨架（1-2周）：
  * 行程创建、列表（本地数据库存储）。
  * 支出记录：核心分类（餐、门、交、住、物、杂）+ 金额 + 时间。
  * 收入记录：核心分类（车销、自费、进店、导服、小费）+ 金额 + 时间 + 备注。
  * 行程总览：总收入、总支出、净收入计算。
  * 数据导出为 CSV

### 总结：这个极简方案的特点
* 死死抓住核心：支出项（餐、门、交...）+ 收入项（车销、自费、进店...）。
* 行程关联：所有收支挂靠到具体团。
* 核心指标突出：总收入、总支出、净收入一目了然。
* 操作极快：3步完成记录，大按钮，离线可用。
* 导游特色鲜明：收入项直接体现车销、自费、进店等导游业务模式。
* 轻量开发：本地存储，无需后端，快速出MVP
